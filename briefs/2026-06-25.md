## Where the puck is going

**Voice agent testing just got its Waymo moment.** Coval's $28M — ex-Waymo evaluation lead at the helm, Twilio Ventures in the cap table — crystallizes what the field has been circling: deploying voice agents at enterprise scale requires simulation-first discipline. YC partner Harj Taggar's tweet this week put it plainly: "Deploying voice agents in the enterprise needs the same testing discipline as self-driving cars." Coval's 60+ customers and last week's 60% demo-to-production failure rate data (seven documented failure modes) confirm that testing infrastructure is where enterprise spend moves next. For builders: if you're selling voice agents without simulation and continuous evaluation, you're selling a self-driving car without a safety driver.

**The satisfaction gap is the opportunity map.** Deepgram's numbers — 80% adoption, 21% satisfaction — distill the market into a single 59-point gap. That's not a quality problem; it's an architecture problem. The same report shows 46% want model fine-tuning. Last week's failure analysis identified seven production failure modes (latency degradation, STT accuracy in noise, model drift, hallucination spikes, telephony failures, state breaks, observability gaps). Each is a product opportunity. The answer isn't better models — it's better testing (Coval), better operations (Retell, Vapi), better governance (Voices for CX), and better deployment tooling. CCW 2026 added Talkdesk's Agent Builder (natural-language creation, pre-deployment validation, continuous governance) to this thesis: the bottleneck has moved from model quality to production reliability.
